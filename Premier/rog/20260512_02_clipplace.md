## 重要な技術知見

### `app.project.getSelection()` は存在しない（確定）

```
ReferenceError: app.project.getSelection is not a function
```

- Premiere Pro の公式 ExtendScript API に `Project.getSelection()` は**存在しない**
- After Effects の `app.project.selection` と混同しやすいので注意
- **結論: プロジェクトパネルの選択状態は ExtendScript から取得不可能**

### 代替手段: `getInsertionBin()` + UI チェックボックス

```javascript
var bin = app.project.getInsertionBin();
// bin = プロジェクトパネルで最後にダブルクリックして開いたビン
// bin.children でビン内の ProjectItem を列挙できる
```

- ビンを開く（ダブルクリック）= 挿入ビンになる
- `bin.children[i].nodeId` でクリップを一意識別
- UI 側でチェックボックス一覧を表示し、ユーザーが選択する設計に

### ProjectItem の種別判定

```javascript
var type = 0;
try { type = child.type; } catch(e) {}
// type 2 = BIN（フォルダ）, type 3 = ROOT → スキップ
// type 1 = クリップ → 対象
if (type !== 2 && type !== 3) { items.push(child); }
```

`ProjectItemType.BIN` 定数が未定義の場合があるため数値で比較する。

### nodeId によるクリップ再探索

ビン内容取得時に `nodeId` を記録し、配置実行時に `rootItem` を再帰探索して ProjectItem を復元。

```javascript
function findByNodeId(item, nodeId) {
    try { if (item.nodeId === nodeId) return item; } catch(e) {}
    try {
        if (item.children) {
            for (var i = 0; i < item.children.numItems; i++) {
                var result = findByNodeId(item.children[i], nodeId);
                if (result) return result;
            }
        }
    } catch(e) {}
    return null;
}
```

### CEP ↔ ExtendScript データのシリアライズ

`JSON.stringify` は ExtendScript 側では環境依存があるため、タブ区切りテキストで安全に受け渡す。

```
"OK:ビン名\tname1\tnodeId1\tname2\tnodeId2..."
```

UI 側 JavaScript でパース:
```javascript
var parts = result.replace('OK:', '').split('\t');
var binName = parts[0];
for (var i = 1; i + 1 < parts.length; i += 2) {
    loadedItems.push({ name: parts[i], nodeId: parts[i+1] });
}
```

### タイムラインの基準クリップ絞り込み

`seq.getSelection()` はタイムライン上の選択クリップを返す（こちらは正常動作）。  
基準トラックのクリップと `start.seconds` を照合して絞り込む。

```javascript
var seqSel = seq.getSelection();
var selectedTimes = {};
for (var s = 0; s < seqSel.length; s++) {
    selectedTimes[seqSel[s].start.seconds.toFixed(6)] = true;
}
// hasTimelineSelection = true のときのみ絞り込む
```

### `isTargeted()` の互換対応

```javascript
function isTrackTargeted(track) {
    try {
        if (typeof track.isTargeted === "function") return track.isTargeted();
        if (typeof track.isTargeted !== "undefined") return !!track.isTargeted;
    } catch(e) {}
    return false;
}
```

バージョンによって関数か否かが異なるため両方チェック（範囲セレクター v2 と同じパターン）。

## 使い方フロー

```
1. Premiere プロジェクトパネルで配置したいクリップのビンをダブルクリック
2. エクステンション「クリップ選択」→「読み込み」ボタン
   → ビン内クリップがチェックボックス付きで一覧表示
3. 配置したいクリップをチェック（すべて選択 or 個別）
4. 基準トラック・配置先トラックを設定
   （「ターゲット取得」で Premiere のターゲット設定から自動入力可）
5. 「配置実行」
```

**配置ルール:**
| 選択件数 | 動作 |
|---------|------|
| 1件 | 基準トラックの全クリップ位置に繰り返し配置 |
| 複数件 | 上から順に基準クリップと 1 対 1 で配置 |

タイムライン上で基準トラックのクリップを選択中の場合は、選択した位置のみに配置される。

```javascript
isTrackTargeted(track)

getTargetedTracks()

findByNodeId(item, nodeId)
// → ProjectItem | null  (rootItem から再帰探索)

placeClipsByNodeIds(nodeIdsStr, refTrackType, refTrackNum, destTrackType, destTrackNum, matchDuration)
// nodeIdsStr    : カンマ区切りの nodeId 文字列
// refTrackType  : 0=ビデオ, 1=オーディオ
// refTrackNum   : 1始まりのトラック番号
// destTrackType : 0=ビデオ, 1=オーディオ
// destTrackNum  : 1始まりのトラック番号
// matchDuration : true=基準クリップ長に合わせる
// → "OK:N件配置" / "ERROR:..."
```
